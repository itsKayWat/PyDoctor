PyDoctor

A comprehensive Python environment diagnostic and repair tool, designed to make Python environment management painless.

Purpose

PyDoctor was created to solve common Python environment issues automatically, saving developers time and frustration when setting up or maintaining Python environments. It provides automated diagnosis and repair capabilities for various Python-related issues.

Key Features

  • Automatic environment diagnosis
  • Package installation verification
  • System resource monitoring
  • Network connectivity checks
  • Automatic repair attempts
  • Detailed logging
  • Environment variable management
  • Package dependency resolution

Installation

  1. Clone the repository: git clone https://github.com/yourusername/PyDoctor.git
  2. Navigate to the project directory: cd PyDoctor
  3. Run the requirements installer: python requirements.py

Requirements

  • Python 3.7 or higher
  • pip (Python package installer)
  • Internet connection for package installation
  • Windows/Linux/MacOS compatible

Usage

  1. Run the diagnostic tool: python src/troubleshoot.py
  2. Review the diagnostic results in the generated log file
  3. Allow the tool to attempt automatic repairs if issues are found
  4. Use the test application to verify functionality: python src/pyqt_test.py

Common Use Cases

Package Installation Issues

When developers experience package import errors or installation failures, PyDoctor can automatically diagnose and repair the installation.

Environment Setup

New developers can use PyDoctor to verify their Python environment setup and fix any issues automatically.

System Resource Issues

When applications perform poorly or crash, PyDoctor can analyze system resources and identify potential bottlenecks.
